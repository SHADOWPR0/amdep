# Robotics Extension

AmDep begins with personnel deployment because people are the expensive and immediate constraint. The same ontology expands into robotics and fleet deployment.

## Robotic Asset Types

- site survey drones
- autonomous layout rovers
- inspection drones
- materials rovers
- security drones
- concrete scan robots
- thermal inspection drones
- progress capture units

## First Use Cases

Start with work that reduces supervisor travel and improves proof:

- roof inspection capture
- storm damage documentation
- progress verification
- safety walk documentation
- thermal inspection
- closeout evidence
- site surveillance
- materials movement pilots

## Deployment Questions

The robotics layer answers:

- which sites are robot-ready
- which asset can operate there
- what proof is required
- what human role is still needed
- what travel is removed
- what inspection or rework cost may be avoided
- which operator approved the deployment

## Product Path

1. Score robotics suitability from job phase, inspection need, urgency, weather, and site readiness.
2. Assign available robots to compatible jobs.
3. Track avoided inspection trips and proof packets.
4. Add site capture evidence.
5. Integrate robotics vendors through a fleet API.
6. Optimize mixed human/robot deployment.

Robotics should not be sold as magic. Sell it as field proof, route reduction, and repeatable capture.
